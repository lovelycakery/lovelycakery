# 版本管理系統使用說明

## 📋 系統簡介

這個版本管理系統使用 `versions.json` 文件記錄所有版本信息，您只需要透過對話告訴我版本編號和說明，我會自動記錄並在需要時幫您恢復。

## 🚀 如何使用

### 1. 創建新版本

**對話範例：**
```
「創建版本 v1.1，說明是：新增日曆功能」
「建立版本 v2.0，說明：重大更新，重新設計首頁」
```

**我會執行：**
- 獲取當前 git commit hash
- 創建 git tag（如 v1.1）
- 更新 `versions.json` 記錄版本信息

### 2. 列出所有版本

**對話範例：**
```
「列出所有版本」
「顯示版本列表」
「有哪些版本？」
「查看所有版本」
```

**我會顯示：**
- 所有版本的編號和說明
- 創建日期
- Git commit 信息

### 3. 恢復到指定版本

**對話範例：**
```
「恢復到版本 v1.0」
「回到版本 v1.1」
「切換到版本 v2.0」
```

**我會執行：**
- 從 `versions.json` 找到對應的 git commit/tag
- 使用 git 恢復所有文件到該版本
- 更新 `current_version` 記錄

### 4. 查看特定版本信息

**對話範例：**
```
「版本 v1.0 的說明是什麼？」
「v1.1 是什麼時候創建的？」
```

## 📝 版本記錄格式

版本信息記錄在 `versions.json` 文件中，格式如下：

```json
{
  "versions": [
    {
      "version": "v1.0",
      "description": "初始版本",
      "git_commit": "完整的 commit hash",
      "git_commit_short": "短 commit hash",
      "git_tag": "v1.0",
      "created_at": "2025-12-25",
      "created_by": "創建者名稱",
      "files_changed": []
    }
  ],
  "current_version": "v1.0"
}
```

## 🎯 完整使用範例

### 範例 1：創建多個版本

```
您：「創建版本 v1.0，說明是：初始版本」
我：✅ 已創建版本 v1.0
   說明：初始版本
   Git commit: d9cabc2

您：「創建版本 v1.1，說明是：新增日曆功能」
我：✅ 已創建版本 v1.1
   說明：新增日曆功能
   Git commit: xxxxxxx

您：「創建版本 v1.2，說明是：優化首頁樣式」
我：✅ 已創建版本 v1.2
   說明：優化首頁樣式
   Git commit: yyyyyyy
```

### 範例 2：查看和恢復版本

```
您：「列出所有版本」
我：📋 版本列表：
   v1.0 - 初始版本 (2025-12-25)
   v1.1 - 新增日曆功能 (2025-12-26)
   v1.2 - 優化首頁樣式 (2025-12-27)

您：「恢復到版本 v1.0」
我：🔄 正在恢復到版本 v1.0...
   ✅ 恢復完成！已將所有文件恢復到 v1.0 版本
```

## ⚠️ 注意事項

1. **版本編號格式**：建議使用 `v1.0`、`v1.1`、`v2.0` 等格式
2. **恢復前確認**：恢復操作會覆蓋當前文件，請確認後再執行
3. **Git 提交**：創建版本時會自動創建 git tag，但不會自動提交當前更改
4. **版本唯一性**：每個版本編號只能使用一次

## 🔧 技術細節

- **版本記錄**：`versions.json` 文件
- **Git 標籤**：每個版本會創建對應的 git tag
- **恢復方式**：使用 `git checkout <commit/tag> -- .` 恢復文件

## 📂 文件位置

- 版本記錄文件：`versions.json`
- 使用說明：`版本管理使用說明.md`（本文件）

---

**提示**：所有操作都透過對話完成，您不需要手動編輯 `versions.json` 或使用 git 命令。
